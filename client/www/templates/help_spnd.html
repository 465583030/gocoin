Spending BTC

Spending money is a secured process that requires several steps and moving files between your online client PC and your offline wallet machine. USB flash disk is probably the most convenient medium to use for this purpose.

Never move any files others than the ones that you actually need to move, which are:
  • The balance/ folder containing your unspent outputs - you move it from client to wallet
  • Text files with signed transactions - these you move from wallet to client
Assuming that you would not sign a wrong transaction, nothing in the files you move between the two points is security sensitive, so you do not need to worry about protecting the medium (USB disk).

1. Export your balance

From the moment when wallet.txt (or its renamed version) is properly loaded into your client, you can check and export your balance. In order to spend your coins you will need to copy the balance folder (containing unspent.txt and one or more <txid>.tx files) to the PC that runs your wallet.
Use Home page to download balance.zip file. Just extract the zip file to get the balance folder.

MakeTx page allows you to use a so-called coin control feature. There you can manually assemble a transaction in quite a convenient and interactive way. After you fill in all the transaction details, it will give you payment.zip with the exact command that you shall execute at the wallet machine.

2. Sign your transaction

If you have payment.zip created via MakeTx page (as opposite to balance.zip), then only need to extract the archive at your wallet machine and execute the exact command from the payment command file. Otherwise follow the steps below.

To spend your money move balance.zip to your wallet machine and extract it. Now when you execute wallet without any parameters (in the same folder where the balnace/ is placed), it should show you how much BTC you are able to spend, from your current balance. To spend your coins, order the wallet to make and sign a new transaction, using a command like:
wallet -send 1JbdKe4eBwtexisGTbCKY5v5CfphtdZXJs=0.01
Note no spaces between the address, the equal sign and the value. The value represents the amount in BTC, which you request to send to the given address.

There are command line switches that you may find useful when running wallet (i.e. a fee control). Run wallet -h to see them all.

3. Verify signed transaction

If everything goes well with the wallet -send ... command, it will create a text file with a signed transaction. The file is named like 01234567.txt - this is the file you need to move to your online node in order to notify the network about your payment, but before doing so, use wallet tool to decode the transaction and double check whether it actually does what you intended:
wallet -d 01234567.txt

If the decoded outputs look OK, move the transaction file to your online node.

4. Broadcast transaction

Press Load TX to open a dialog where you should paste the content of the signed transaction file. After pressing OK, you will be taken to the Transactions page where the details of your transaction will be shown. Verify them again, this time also making sure that the fee is not too big.

Re-broadcasting transactions

The client never broadcasts transactions unrequested, so if your transaction does not get confirmed soon enough, you may want to re-broadcast it, using the same method as for the initial broadcasting. There may of course be other reasons why your transaction does not get confirmed (usually because the fee was to low), in which case re-broadcasting it will not help you much.
